# 02 – Sequences and Collections
 
> [!abstract] Overview
> Python's generic data structures: **list**, **tuple**, **dictionary**, **set**, and more. All are containers of *references* to objects — Python never stores values directly.
 
---
 
## Lists
 
> [!info] What is a `list`?
> A **mutable**, sequential data structure implemented as a variable-length array of references with exponential over-allocation for performance. Replaces arrays, linked lists, vectors, etc. from C++/Java.
 
### Creation & Type Checking
 
```python
l = ['one', 'two', 'three']
type(l)                             # <class 'list'>
isinstance(l, list)                 # True
isinstance(l, (list, tuple))        # True — OR check
isinstance(list, object)            # True — object is root class
```
 
### Adding Elements
 
```python
l.append('four')                    # append one item at end
l.insert(2, 'more')                 # insert at index 2
l = l + ['five', 'six']             # concatenation → new list
['zero'] + l                        # prepend via concat
l.extend([7, 8, 9])                 # append multiple items
l * 2                               # repetition → new list
```
 
### Copying
 
```python
l1 = l[:]          # slice copy (shallow)
l2 = l.copy()      # .copy() method (shallow)
l3 = list(l)       # constructor copy (shallow)
```
 
> [!warning] Name Shadowing
> `list = [1,2,3]` **destroys** the built-in `list` class — a class name is just a reference that can be reassigned.
 
### Generic Containers
 
```python
lst = [123, 4.5, ['hi'], 6-7j, 'wow', (8,9), False, 0xFF]
a, b, c, d, e, f, g, h = lst       # unpacking all elements
```
 
Lists hold **any mix of types** — they only store references.
==This is why we don't need templates in Python.==
 
### Nested Lists & Indexing
 
```python
nst = [1, [3.5, 'hi', 2-3j, ['nothing', False]], 0xFFFF]
nst[1][-1][0]                       # → 'nothing'
 
mtx = [[11,12,13],
       [21,22,23],
       [31,32,33]]
mtx[1][2]                           # → 23
```

note: there is numpy for matrix but not included.
 
### Slicing & Assignment
 
```python
l[3] = 'None'                       # replace one item
l[2:4] = [3, 33, 333]               # replace a slice (can change length!)
del l[-1]                           # remove last element
del l[0:2]                          # remove a slice
 
lx = list(range(12))
lx[2:6] = 'oops'                    # string unpacked as iterable → ['o','o','p','s']
lx[2:6] = ['oops']                  # replaces slice with single-item list
lx[0:4] = []                        # remove slice (same as del lx[0:4])
lx[:] = []                          # clear the list
```
 
> [!warning] Subtle Difference
> `lx[1] = ['a','aa','aaa']` → one item replaced by a **nested list**
> `lx[1:2] = ['a','aa','aaa']` → slice replaced by **three items**
 
### Comparison
 
```python
[1, 2, 3] < [1, 2, 4]              # True — lexicographic (recursive)
(1, 2, 3) < (1, 2, 3, 4)           # True — shorter is "less"
l == l2                             # compares values
l is l2                             # compares identity (references)
```
 
### Sorting & Removing
 
```python
sl = sorted(rl)                     # → new sorted list
rl.sort()                           # sort in place (returns None!)
sorted(bl, reverse=True)            # descending
sorted(bl, key=len)                 # custom key function
 
bl.remove('hi')                     # remove by value (first occurrence)
rl.pop(3)                           # remove by index, returns the item
rl.count(4)                         # count occurrences
```
 
> [!danger] Remember
> `.sort()` returns **`None`** — it sorts in place. Use `sorted()` to get a new list.
 
### References (Aliasing)
 
```python
a = [1, 2, 3]
b = a              # b points to SAME object
a.append(4)
print(b)           # → [1, 2, 3, 4]
 
b = a[:]           # b is now a COPY
a.append(5)
print(b)           # → [1, 2, 3, 4] — unaffected
```
 
### List as Stack / Queue
 
```python
# Stack (fast — O(1) amortised)
nl.append(6)       # push
nl.pop()           # pop (returns item)
nl[-1]             # top / peek
 
# Queue (slow — O(n) for pop(0))
nl.append(8)       # enqueue
nl.pop(0)          # dequeue
```
 
> [!tip] Use `collections.deque` for Efficient Queues
> ```python
> from collections import deque
> q = deque(['Eric', 'John', 'Mike'])
> q.append('Toby')      # enqueue
> q.popleft()            # dequeue — O(1)
> ```
> Implemented as a linked-list of blocks of pointers.
 
### Heapq & Merge Sort
 
```python
from heapq import merge, nlargest
nlargest(3, [4,3,6,2,8,0,7,1,9,3])    # → [9, 8, 7]
list(merge([1,3,5,7], [0,2,4,8]))      # → [0,1,2,3,4,5,7,8]
 
# Merge sort — imperative style
def merge_sort(a):
    if len(a) <= 1: return a
    mid = len(a) // 2
    return merge(merge_sort(a[:mid]), merge_sort(a[mid:]))
 
# Merge sort — FP one-liner
def merge_sort(a):
    return a if len(a) <= 1 else merge(merge_sort(a[:len(a)//2]), merge_sort(a[len(a)//2:]))
```

## Tuples
 
> [!info] What is a `tuple`?
> A **lightweight, immutable list**. Fewer methods but better performance — memory allocation is optimised internally. Used heavily *inside* Python for parallel assignment, multiple return values, etc.
 
### Creation & Unpacking
 
```python
t = (1, 2, 3)                      # tuple literal
u = 2, 4, 6                        # packing (parentheses optional)
x, y, z = t                        # unpacking
 
u = (2,)                            # singleton — comma required!
u = ('two',)                        # without comma → just a string
```
 
### Tuples for Multiple Return Values
 
```python
def sqpair(x): return x, x**2      # returns a tuple
v, v2 = sqpair(7)                   # unpacked: v=7, v2=49
```
 
### Conversion
 
```python
lt = list(t)                        # tuple → list
tt = tuple(lt)                      # list → tuple
```
 
### Named Tuples (Immutable Structs)
 
```python
from collections import namedtuple
Point = namedtuple('Point', ['x', 'y'])
 
pA = Point(2, 7)
pA.x                                # access by name
px, py = pA                         # still unpackable
# pA.x = 5                          # AttributeError — immutable
```
 
> [!quote] Design Rule
> **"Tuples have structure, lists have order."**
> Tuples = heterogeneous, fixed records (name, ID, coordinates).
> Lists = homogeneous, ordered collections (list of names, list of points).
 
### Mutability Nuance
 
```python
tt = ([1,2,3], [4,5], [6,7,8], [9])
# tt[1] = [444,2,1]                 # TypeError — can't reassign reference
tt[1][0] = 444                      # OK — mutating the list INSIDE the tuple
tt[1][:] = ['tada', 'voila']        # OK — replacing list contents
```
 
> [!warning]
> A tuple's *references* are immutable, but the *objects they point to* can still be mutable.

## Dictionaries
 
> [!info] What is a `dict`?
> A **mutable associative array** (hash table with open addressing). Stores **key→value** pairs. ==Ordered by insertion since Python 3.7.==
 
### Creation & Access
 
```python
d = {'fr': 'France', 'uk': 'United Kingdom', 'it': '???'}
d2 = dict(aye=1, bee=2, sea=3)     # keyword syntax for string keys
 
d['fr']                             # key lookup (KeyError if missing)
d.get('fr')                         # same, but returns None if missing
d.keys()                            # dict_keys view
d.values()                          # dict_values view
d.items()                           # dict_items view (key, value) pairs
```
 
### Iteration
 
```python
for k in d: print(k)               # iterate over keys (default)
for k in d.keys(): print(k)        # same, explicit
for v in d.values(): print(v)      # iterate over values
for k, v in d.items(): print(k, v) # iterate over pairs
```
 
### Modification
 
```python
d['it'] = 'Italy'                   # update value
d['ch'] = 'Switzerland'             # add new pair
del d['uk']                         # remove pair
d[42] = 'yes'                       # keys can be any immutable type
```
 
### Unpacking & Flattening
 
```python
x, y, z = {1:'a', 2:'b', 3:'c'}    # unpacks keys
print(*d)                           # flatten → prints keys
print([*d])                         # keys as list
```
 
### Hashing
 
```python
hash(1)                 # int hash
hash('1')               # string hash
hash((1, 2))            # tuple hash
# hash([1, 2])          # TypeError — lists are unhashable
```
 
> [!note] Key Requirement
> Only **immutable** (hashable) objects can be dictionary keys: `int`, `str`, `tuple`, `frozenset`, etc.
 
### Ordering & OrderedDict
 
Since Python 3.7, `dict` preserves **insertion order**. `OrderedDict` from `collections` is nearly redundant but offers extras like `move_to_end()`:
 
```python
from collections import OrderedDict
od = OrderedDict([('z', 26), ('g', 7), ('k', 11)])
od.move_to_end('z')
```
 
Equivalent with regular dict:
```python
dd['z'] = dd.pop('z')              # move key to end
```

## Sets
 
> [!info] What is a `set`?
> A **mutable, unordered** collection of **unique, hashable** items. Backed by a hash table.
 
### Creation
 
```python
s1 = {'France', 'United Kingdom', 'Italy', 'Germany'}
s2 = set([4,7,1,9,4,8,2,1])        # duplicates removed automatically
```
 
> [!tip] Remove Duplicates from a List
> ```python
> list(set(alist))
> ```
 
### Set Operations
 
| Operation          | Operator | Method                |
| ------------------ | -------- | --------------------- |
| Intersection       | `&`      | `.intersection()`     |
| Union              | `\|`     | `.union()`            |
| Difference         | `-`      | `.difference()`       |
| Symmetric Diff     | `^`      | `.symmetric_difference()` |
| Subset             | `<=`     | `.issubset()`         |
| Intersection Update | `&=`    | `.intersection_update()` |
| Union Update       | `\|=`   | `.update()`           |
 
```python
s3, s4 = set('pecan'), set('pie')
s3 & s4        # → {'e', 'p'}
s3 | s4        # → {'a', 'c', 'e', 'i', 'n', 'p'}
s3 - s4        # → {'a', 'c', 'n'}
s3 ^ s4        # → {'a', 'c', 'i', 'n'}
```
 
### Frozen Set (Immutable)
 
```python
s5 = frozenset(s4)
# s5.add('x')                       # AttributeError
```

## Other Collections
 
### Counter (Bag)
 
```python
from collections import Counter
c = Counter('abcdeabcdabcaba')
c['a']                              # count of 'a'
isinstance(c, dict)                 # True — subclass of dict
c.most_common(3)                    # top 3 elements
sorted(c.elements())                # all elements with repetitions
```
 
### Data Structure Comparison
 
| Python          | Java           | C++                 |
| --------------- | -------------- | ------------------- |
| `list` / `tuple`| `Vector`, `ArrayList` | `vector` / `array` |
| `list`          | `LinkedList`   | `forward_list` / `list` |
| `list` (stack)  | `Stack`        | `stack`             |
| `list` (queue)  | `Queue`        | `queue`             |
| `deque`         | `Deque`        | `deque`             |
| `dict`          | `HashMap`      | `unordered_map`     |
| `OrderedDict`   | `TreeMap`      | `map`               |
| `set`/`frozenset`| `Set`         | `unordered_set`     |
| `PriorityQueue` | `PriorityQueue`| `priority_queue`    |
 
### Immutable vs Mutable Types
 
- **Immutable:** `bool`, `bytes`, `int`, `float`, `Fraction`, `complex`, `str`, `tuple`, `namedtuple`, `frozenset`, `range`, `Counter`
- **Mutable:** `list`, `dict`, `set`, `deque`, `heapq`, `ChainMap`, `PriorityQueue`

## Lisp/Scheme Emulation
 
Cons cells via nested tuples:
 
```python
lisp_list = ("like", ("this", ("example", None)))
 
def car(lst): return lst[0]         # first element
def cdr(lst): return lst[1]         # rest of list
def cons(atm, lst): return (atm, lst)
 
car(cdr(lisp_list))                 # → 'this'
cons('just', lisp_list)             # → ('just', ('like', ...))
```

## Key Takeaways
 
> [!success] Exam Essentials
> - `.sort()` returns ==`None`== — use `sorted()` for a new list
> - `l[:]` creates a **shallow copy**
> - `==` compares **values**, `is` compares **identity** (references)
> - `l[1] = [...]` nests a list; `l[1:2] = [...]` splices items in
> - Only **immutable** types can be dict keys / set members
> - Duplicate check: `len(t) != len(set(t))`
> - Tuples have **structure**, lists have **order**
