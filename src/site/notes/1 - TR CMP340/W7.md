#### Hashing  
Assume we have a set of n values that we want to store in an array of size m<n.  
We can use a function h(k) = k mod m (the hashing function) that takes a value k and outputs an index i for inserting (or finding) k in the list.  
Example: Assume we want to store the numbers 3, 7, 4, 9, 1, and 6 in an array of size 4


> [!success]- Solution
> - We have 6 values: 3, 7, 4, 9, 1, 6
> - We have an array of size **m = 4** (indices 0–3)
> - Hash function: h(k) = k mod 4
> 
> **How it works:**
> Each value gets mapped to an index using mod 4:
> - h(3) = 3 mod 4 = **3** → goes to slot 3
> - h(7) = 7 mod 4 = **3** → also slot 3 (collision!)
> - h(4) = 4 mod 4 = **0** → goes to slot 0
> - h(9) = 9 mod 4 = **1** → goes to slot 1
> - h(1) = 1 mod 4 = **1** → also slot 1 (collision!)
> - h(6) = 6 mod 4 = **2** → goes to slot 2
> 
> When two values hash to the same slot (a **collision**), they get chained together in a linked list. So the array looks like:
> - Slot 0: → [4]
> - Slot 1: → [9] → [1]
> - Slot 2: → [6]
> - Slot 3: → [3] → [7]
> 
> **Complexity:**
> - **T(m,n) = O(n/m)** — average. we distribute n elements over m buckets. So searching within a list of n/m lists in extreme case n/m iterations.
> - O(n) worst case
> - O(1) best case 1 element in each bucket. 
> - **S(m,n) = O(m + n)** — m slots for the array + n total elements stored across all chains

void Heapify(int A[], int n, int i) {
    if (i >= n) return;
    int left = 2 * i + 1;
    int right = 2 * i + 2;
    Heapify(A, n, left);
    Heapify(A, n, right);
    if (left < n && A[left] >= A[i] && (right >= n || (right < n && A[left] >= A[right]))) {
        int x = A[i];
        A[i] = A[left];
        A[left] = x;
        siftdown(A, n, left);
    }
    else if (right < n && A[right] >= A[i] && A[right] >= A[left]) {
        int x = A[i];
        A[i] = A[right];
        A[right] = x;
        siftdown(A, n, right);
    }
}

void siftdown(int A[], int n, int i) {
    if (i >= n) return;
    int left = 2 * i + 1;
    int right = 2 * i + 2;
    if (left < n && A[left] >= A[i] && (right >= n || (right < n && A[left] >= A[right]))) {
        int x = A[i];
        A[i] = A[left];
        A[left] = x;
        siftdown(A, n, left);
    }
    else if (right < n && A[right] >= A[i] && A[right] >= A[left]) {
        int x = A[i];
        A[i] = A[right];
        A[right] = x;
        siftdown(A, n, right);
    }
}

###### Stable vs Unstable Sort
![image-131.png|400x421](/img/user/1%20-%20TR%20CMP340/img/image-131.png)

**Stable:**
- Bubble sort — only swaps adjacent elements, equal elements never cross each other
- Insertion sort — shifts elements one by one, equal elements stay in order
- Merge sort — when merging, if left and right are equal, it picks left first
- Counting sort — processes elements in order, preserves original positions
- Radix sort — uses counting sort as a subroutine, so inherits stability
**Unstable:**
- Selection sort — swaps minimum with a distant position
- Quicksort — partitioning swaps elements over long distances
- Heap sort — extraction swaps root with last element
